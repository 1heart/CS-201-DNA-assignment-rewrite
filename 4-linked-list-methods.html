LinkStrand Methods Overview

Because LinkStrand implements the IDnaStrand interface, all the methods you are overriding are documented in comments, so you should check there if you are wondering how any methods should work. You can also refer to the working methods in SimpleStrand, although they will differ in implementation because they use a StringBuilder instead of a linked list.

    Constructor and initializeFrom:
  • Implement both with the same code (either copy the code or re-use the common code in one method.
  • When creating/initializing a new LinkStrand only one node should be created. This node represents the entire string representation of DNA.
  • Be sure to initialize the length of the simulated strand.
    Append:
  • Implementation should not concatenate strings, but create new nodes
  • If youâ€™re appending a LinkStrand you should create new nodes you should not convert the strand to a string.
  • If you're appending another type of Strand, ie. a SimpleStrand, you should convert the strand to a string and then append it, you can refer to the implementation in Simple Strand. (Hint: the instanceof operator could be helpful)
    CutandSplice:
  • You may assume there is only one node (though it may contain a huge String of DNA). If there is more than one node throw an exception, ex. if(myFirst.next != null)) {throw new RuntimeException("link strand has more than one node")};
  • The implementation of this method should be virtuall identical to the code in SimpleStrand
